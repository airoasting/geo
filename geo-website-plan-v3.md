# GEO 7-Pillar 브랜드 평가 웹사이트 — 구축 계획서 v3

> **목표:** 브랜드 3개를 입력하면 Claude API(사용자 Max 키)가 GEO 7-Pillar을 자동 평가하고, 스네이크 차트로 비교 결과를 보여주는 웹사이트. 진단 결과는 개별 페이지로 저장되어 히스토리로 관리된다.

---

## 1. 사이트 구조

```
메인 (index.html)
 ├── [1] 프레임워크  ──→ framework.html         ← GEO 7-Pillar 설명
 ├── [2] 진단하기    ──→ diagnose.html          ← 브랜드 3개 + 산업군 입력 → API 호출
 └── [3] 히스토리    ──→ history.html           ← 과거 진단 결과 목록
                          └── 결과 상세  ──→ result.html?id=xxx  ← 개별 결과 페이지
```

### 2-1. 프레임워크 (framework.html)

GEO 7-Pillar을 **처음 보는 사람이 이해**할 수 있도록 설명하는 페이지.
**API 키 없이 접근 가능** — 누구나 프레임워크를 먼저 이해할 수 있다.

| 섹션 | 내용 |
|------|------|
| Hero | "브랜드가 AI에게 선택받는 7가지 기준" |
| 4-Layer 구조도 | Gate → Core → Value → Risk 시각화 |
| 7-Pillar 아코디언 | **기본:** 항목 이름 + 핵심 질문 + 태그라인 비유 |
|  | **클릭 확장:** 평가 포인트 3개 + 상세 비유 + 경계 설명 |
| 점수 체계 설명 | GEO 총점(10점) vs ARS(100점) 차이 + Veto 규칙 |
| 데모 결과 | 스타벅스 vs 블루보틀 vs 이디야 샘플 스네이크 차트 (API 호출 없이 하드코딩) |
| CTA | "지금 진단해보기 →" |

### 2-2. 진단 페이지 (diagnose.html)

**핵심 흐름:** API 키 확인 → 브랜드 3개 + 산업군 입력 → Claude API 병렬 호출 → 결과 페이지로 이동

```

┌──────────────────────────────────┐
│  🔑 Claude API 키가 필요합니다    │
│                                  │
│  API 키 발급 방법:                │
│  1. console.anthropic.com 접속   │
│  2. Settings → API Keys          │
│  3. "Create Key" 클릭 후 복사     │
│  [자세한 안내 보기 →]             │
│                                  │
│  API Key: [sk-ant-...        ]   │
│  [ ] 이 브라우저에 키 저장        │
│                                  │
│  ※ 키는 브라우저에만 저장되며     │
│    서버로 전송되지 않습니다        │
│                                  │
│         [ 확인 ]                  │
└──────────────────────────────────┘
  ※ 이미 저장된 키가 있으면 이 단계 건너뜀
         │
         ▼
[Step 1] 입력 (총 4개 필드 — 모두 필수)
┌──────────────────────────────────┐
│                                  │
│ 브랜드 A: [스타벅스        ] *   │
│ 브랜드 B: [블루보틀        ] *   │
│ 브랜드 C: [이디야          ] *   │
│ 산업군:   [음식료품       ▾] *   │
│                                  │
│  ※ "기타" 선택 시 직접 입력 표시  │
│  기타 입력: [______________ ] *  │
│                                  │
│        [ 🔍 AI 평가 시작 ]       │
│                                  │
│  ※ 4개 필드 모두 입력해야         │
│    버튼이 활성화됩니다            │
└──────────────────────────────────┘
```

#### 입력 유효성 검증

| 규칙 | 동작 |
|------|------|
| **4개 필드 모두 필수** | 하나라도 비어 있으면 "AI 평가 시작" 버튼 비활성화 (disabled, 회색 처리) |
| **버튼 활성화 조건** | 브랜드 A, B, C 모두 1자 이상 + 산업군 선택 완료 (기타 선택 시 텍스트도 입력) |
| **실시간 검증** | 입력할 때마다 체크 — 4개 충족 시 버튼 즉시 활성화 (초록색) |
| **빈 필드 안내** | 비활성 버튼 호버 시 "브랜드 3개와 산업군을 모두 입력해주세요" 툴팁 |
| **브랜드 중복 체크** | 같은 이름 2개 이상 입력 시 "브랜드명이 중복됩니다" 경고 (진행은 허용) |
| **공백만 입력 방지** | trim() 후 빈 문자열이면 미입력으로 처리 |
         │
         ▼
[Step 2] 로딩 (Claude API 병렬 호출)
┌──────────────────────────────────┐
│  ⏳ AI가 브랜드를 분석하고 있습니다  │
│                                  │
│  🔄 스타벅스 분석 중...           │
│  🔄 블루보틀 분석 중...           │
│  🔄 이디야 분석 중...             │
│                                  │
│  ※ 3개 브랜드 동시 분석 (30초 내외) │
└──────────────────────────────────┘
         │
         ▼
  완료 시 result.html?id=xxx 로 자동 이동
```

## 5. API 연동 상세

### API 키 관리 (BYOK: Bring Your Own Key)

```
사용자 흐름:
1. 진단 페이지 최초 진입 → "API 키를 입력하세요" 모달
2. 사용자가 자신의 Claude Max API 키 입력
3. "이 브라우저에 키 저장" 체크 시 localStorage에 저장
4. 다음 방문부터 저장된 키 자동 사용
5. 설정에서 키 변경/삭제 가능

보안 원칙:
- 키는 localStorage에만 저장, 서버/프록시에 저장하지 않음
- 프록시는 키를 패스스루(pass-through)만 함
- 키는 브라우저 개발자도구에서 확인 가능하다는 안내 표시
```

### CORS 프록시 (Cloudflare Workers)

```
브라우저 (fetch)
    │
    ▼
Cloudflare Worker (geo-proxy.workers.dev)
    │  - 요청 헤더에서 API 키 추출
    │  - Anthropic API로 전달
    │  - 응답을 그대로 반환
    │  - 키를 저장/로깅하지 않음
    ▼
Anthropic API (api.anthropic.com/v1/messages)
```

### 시스템 프롬프트 (Claude에게 전달)

```
당신은 GEO 7-Pillar 브랜드 평가 전문가입니다.

[GEO 7-Pillar v2.1 정의 전문 삽입]

사용자가 브랜드명과 산업군을 제공하면:
1. web_search를 사용하여 해당 브랜드의 최신 정보를 수집하세요
2. 7개 Pillar 각각을 10점 만점으로 평가하세요
3. 해당 브랜드가 AI 추천 상위에 올라올 수 있는 대표 검색 질문 예시 3개를 생성하세요
4. 7점 이하인 Pillar에 대해 구체적 개선 액션 아이템을 2개씩 제시하세요
5. 반드시 아래 JSON 형식으로만 응답하세요 (다른 텍스트 없이 JSON만)

평가 원칙:
- 검증 가능한 사실에 기반하여 채점
- 각 점수에 한 줄 근거 필수
- 산업 특성을 고려한 상대 평가
- 추천 질문은 해당 브랜드의 강점/USP가 잘 드러나는 실제 사용자 질문 형태로 작성
- 개선 액션은 "누가, 무엇을, 어디에" 수준으로 구체적으로 작성
- 액션은 1개월 이내에 시작 가능한 것만 제시
- 추상적 표현 금지 (❌ "브랜드 인지도를 높이세요" → ✅ "공식 사이트에 JSON-LD Product 스키마 추가")
- 확실한 근거가 없으면 중간 점수(5~6)를 부여하고, 동일 기준을 일관되게 적용하세요

응답 형식:
{
  "brand": "브랜드명",
  "industry": "산업군",
  "pillars": {
    "discoverability":  { "score": 0, "reason": "" },
    "source_authority": { "score": 0, "reason": "" },
    "info_depth":       { "score": 0, "reason": "" },
    "recency":          { "score": 0, "reason": "" },
    "consistency":      { "score": 0, "reason": "" },
    "distinctiveness":  { "score": 0, "reason": "" },
    "safety":           { "score": 0, "reason": "" }
  },
  "recommended_queries": [
    "이 브랜드가 AI 추천 상위에 올라올 수 있는 검색 질문 예시 3개"
  ],
  "action_items": [
    {
      "pillar": "7점 이하인 Pillar 키",
      "score": 0,
      "actions": [
        "구체적 개선 액션 1",
        "구체적 개선 액션 2"
      ]
    }
  ]
}
```

> **GEO 총점과 ARS는 API 응답에 포함하지 않음.** 프론트엔드의 calculator.js에서 계산. AI의 계산 실수를 방지하고 일관성을 보장.

### API 호출 흐름

```
사용자 입력 (브랜드 3개 + 산업군)
        │
        ▼
  Promise.all([           ← 3건 병렬 호출 (총 15~30초)
    callAPI(브랜드 A),
    callAPI(브랜드 B),
    callAPI(브랜드 C),
  ])
        │
        ▼
  각 응답 JSON 파싱
        │
        ▼
  프론트에서 GEO 총점 + ARS 계산 (calculator.js)
        │
        ▼
  고유 ID 생성 (timestamp 기반)
        │
        ▼
  localStorage에 결과 저장
        │
        ▼
  result.html?id=xxx 로 자동 이동
```

### 에러 처리

| 상황 | 처리 |
|------|------|
| API 키 없음 | 진단 페이지 진입 시 키 입력 모달 (발급 안내 포함) |
| API 키 무효 | "API 키가 유효하지 않습니다. 확인 후 다시 입력해주세요" |
| 네트워크 오류 | 재시도 버튼 + 오류 메시지 |
| **API 타임아웃** | **90초 타임아웃 설정. 초과 시 "분석에 시간이 오래 걸리고 있습니다. 재시도해주세요" + 재시도 버튼** |
| JSON 파싱 실패 | 해당 브랜드만 1회 재호출, 실패 시 "평가 실패" 표시 |
| 브랜드 미인식 | "브랜드 정보를 충분히 찾지 못했습니다" + 낮은 점수로 표시 |
| 일부만 성공 | 성공한 브랜드만 차트에 표시, 실패 브랜드 "재시도" 버튼 |
| **API 사용량 초과** | **"요청이 너무 많습니다. 잠시 후 다시 시도해주세요" (CF Workers rate limit)** |
| **결과 URL 데이터 없음** | **result.html?id=xxx 접근 시 해당 데이터가 localStorage에 없으면: "이 결과는 다른 브라우저에서 생성되었거나 삭제되었습니다. [새 진단 시작] [히스토리로 이동]" 안내** |
| **action_items 누락** | **score ≤ 7인데 action_items가 비어있으면, Pillar별 사전 정의된 기본 액션으로 대체** |

## 6. 점수 계산 로직 (calculator.js)

### GEO 총점

```
GEO 총점 = (P1+P2+P3+P4+P5+P6+P7) ÷ 7    (소수점 1자리)
```

### ARS (AI Recommendation Score)

```javascript
// Phase 1: Veto Check
if (P1 <= 3 || P7 <= 3) {
  arsMax = 20;      // Hard Veto
} else if (P1 <= 5 || P7 <= 5) {
  arsMax = 50;      // Soft Veto
} else {
  arsMax = 100;     // No Veto
}

// Phase 2: Weighted Ranking
arsRaw = ((P2 + P6) * 1.5 + (P1 + P3 + P4 + P5 + P7) * 1.0) / 80 * 100;

// 최종
ars = Math.min(arsRaw, arsMax);
```

### ARS 등급

| 등급 | 점수 | 의미 |
|------|------|------|
| 🟩 S | 91~100 | AI 최우선 추천 |
| 🟦 A | 81~90 | Top 3 안정적 추천 |
| 🟨 B | 71~80 | 일반적 신뢰도 |
| 🟧 C | 51~70 | 정보 존재하나 매력 부족 |
| 🟥 D | 0~50 | 리스크 또는 인식 결여 |

### 개선 우선순위 로직

```
1순위: Gate Layer(①) 또는 Risk Layer(⑦)에 Veto 경고가 있으면 최우선 표시
2순위: 나머지 Pillar 중 최저 점수부터 오름차순 정렬
각 Pillar에 사전 정의된 개선 팁 3개 표시
```
